# AI_GUIDELINES

## Regras para trabalhar neste projeto

- Entender `Program.cs`, `ContaStore` e serviços antes de alterar comportamento.
- Nunca inserir tokens, chat id, senhas ou chaves em arquivos versionados.
- Atualizar documentação após qualquer tarefa relevante.
- Manter arquitetura simples e consistente com Minimal API.
- Evitar duplicação entre frontend e backend.
- Seguir SOLID quando criar novos serviços.
- Priorizar código limpo e validações claras.
- Rodar `dotnet build` antes de finalizar mudanças de código.
- Criar commits pequenos e organizados quando solicitado.
- Explicar mudanças importantes e mostrar `git status` ao finalizar.
